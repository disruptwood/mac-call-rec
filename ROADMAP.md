## Roadmap

### Работает сейчас (v0.3)

- Запись mic (ffmpeg WAV) + system (ScreenCaptureKit WAV)
- Нормализация mic (85% input + loudnorm) + микс → recording.m4a
- Пауза/resume (пробел), стоп (q)
- Транскрипция: Qwen3-ASR 1.7B на MLX (1.6x real-time)
- Speaker labels: Silero VAD на раздельных треках → Я / Собеседни:ца
- 64 теста, source tracks сохраняются

### Этап 1: Streaming транскрипция (prob-20260403-008)

**Цель:** транскрипт появляется в реальном времени во время записи.

**Архитектура (из ресёрча production систем):**
```
mic (sounddevice PCM 16kHz) → Silero VAD gate → когда речь → feed_audio → "Я: ..."
system (capture --pipe PCM)  → Silero VAD gate → когда речь → feed_audio → "Собеседни:ца: ..."
```

**Ключевые принципы:**
- VAD как gate ПЕРЕД моделью — тишина не подаётся, нет галлюцинаций
- Один Qwen3-ASR Session — чанки от mic и system подаются по очереди
- Запись в файлы идёт параллельно и независимо
- Streaming crash не должен влиять на запись

**Pre-conditions:**
- [ ] Протестировать feed_audio с VAD-filtered PCM (не raw)
- [ ] Проверить что sounddevice + ffmpeg могут читать mic одновременно без потери качества
- [ ] Проверить формат PCM из capture --pipe (stereo float32 48kHz → mono int16 16kHz)

### Этап 2: Заметка к записи

Сразу после остановки — ввести описание встречи.
Сохраняется в manifest.json, попадает в заголовок transcript.md.

### Этап 3: Редактирование транскрипта через Claude

Claude CLI вычищает артефакты распознавания, исправляет пунктуацию.
Сохраняет структуру Я/Собеседни:ца и таймстемпы.
Сырой → transcript_raw.md, отредактированный → transcript.md.

### Этап 4: Анализ сессий с Claude

Суммаризация, ключевые темы, трекинг прогресса между сессиями.

### Backlog

- **Оптимизация формата аудио** (prob-20260403-004): system.wav 700 MB/час, можно сжать
- **VAD misattribution**: крупные чанки содержат обоих спикеров, граница нечёткая
- **Возобновляемая транскрипция**: сохранять промежуточные результаты
- **Автоматическая очистка**: удалять source tracks после подтверждения transcript
- **Интеграция в один запуск**: запись + streaming в одном процессе

### Критические правила

1. **Дорожки не должны разъезжаться** — никакой ротации/сегментации во время записи
2. **Source tracks не удалять** — нужны для VAD и пере-транскрипции
3. **E2e тест перед каждой реальной сессией** — запись 1 мин + транскрипция + проверка
4. **VAD gate перед моделью** — без VAD → галлюцинации на тишине
5. **Один model load** — никогда два экземпляра Qwen3-ASR
