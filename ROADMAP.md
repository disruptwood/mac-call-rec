## Roadmap

### Работает сейчас (v0.4)

- Запись mic (ffmpeg WAV) + system (ScreenCaptureKit WAV)
- Нормализация mic (85% input + loudnorm) + микс → recording.m4a
- Пауза/resume (пробел), стоп (q)
- Транскрипция: Qwen3-ASR 1.7B на MLX с integrated pyannote diarization
- Enrolled voice profiles (`~/.call-recorder/speakers/*.npz`) — загрузил один раз, переиспользуется всегда
- Speaker attribution: word-level через pyannote diarization → match к enrolled профилям по cosine similarity
- Фильтрация профилей по сессии (`--speakers "Илья,Мария"`) — не загружать тех кого нет
- Контекстный prompt для ASR (терапия, имена, термины)
- 64 теста, source tracks сохраняются

### Следующий этап: CLI и post-hoc автотранскрипция

**Цель:** запускать запись одной короткой командой с выбором типа сессии стрелками.

Блокер: сначала прогнать обе batch-транскрипции (Мария + Света) и проверить что результат корректный (правильное разделение спикеров, осмысленный текст, корректные имена из enrolled профилей). Только потом трогать CLI.

**Этап A: Короткая CLI с TUI-выбором**

```
$ rec                     # короткий алиас (или `cr`, или что удобно)
   ┌─ Выбери тип сессии ─┐
   │ ▶ Терапия с Марией │
   │   Терапия со Светой │
   │   Созвон            │
   │   Интервью          │
   │   Другое...         │
   └─────────────────────┘
   [↑↓ выбор, Enter старт, Esc отмена]

   Recording... [пробел - пауза, q - стоп]
```

- Типы сессий в конфиге: имя, label для папки, enrolled-спикеры (Илья+Мария / Илья+Света / Илья+имя_собеседника), контекстный prompt override
- После `q`: показать путь к записи, спросить "запустить транскрипцию сейчас? [Y/n]"
- TUI: `textual` (уже попсовая) или `prompt_toolkit`, или простой raw-terminal со стрелками (есть код в cli.py)

**Этап B: Auto-transcribe после записи**

- После stop → автоматически вызвать `scripts/transcribe.py` с правильными `--speakers` и `--context`
- Прогресс в том же терминале
- Если хочешь запись без транскрипции — флаг `--no-transcribe`

**Этап C (opt): Streaming транскрипция во время записи**

- Live-транскрипт сбоку/внизу терминала
- Архитектура из v0.3-планов (mic+system → VAD gate → feed_audio)
- Batch transcribe всё равно прогоняется после остановки (точнее, и даёт диаризацию)
- Опциональный, не делать пока не убедимся что batch работает идеально

### Legacy план: Streaming транскрипция (prob-20260403-008)

_Заморожено до решения по этапу C. Может быть заменено на batch-после-записи._

**Цель:** транскрипт появляется в реальном времени во время записи.

**Архитектура (из ресёрча production систем):**
```
mic (sounddevice PCM 16kHz) → Silero VAD gate → когда речь → feed_audio → "Я: ..."
system (capture --pipe PCM)  → Silero VAD gate → когда речь → feed_audio → "Собеседни:ца: ..."
```

**Ключевые принципы:**
- VAD как gate ПЕРЕД моделью — тишина не подаётся, нет галлюцинаций
- Один Qwen3-ASR Session — чанки от mic и system подаются по очереди
- Запись в файлы идёт параллельно и независимо
- Streaming crash не должен влиять на запись

**Pre-conditions:**
- [ ] Протестировать feed_audio с VAD-filtered PCM (не raw)
- [ ] Проверить что sounddevice + ffmpeg могут читать mic одновременно без потери качества
- [ ] Проверить формат PCM из capture --pipe (stereo float32 48kHz → mono int16 16kHz)

### Этап 2: Заметка к записи

Сразу после остановки — ввести описание встречи.
Сохраняется в manifest.json, попадает в заголовок transcript.md.

### Этап 3: Редактирование транскрипта через Claude

Claude CLI вычищает артефакты распознавания, исправляет пунктуацию.
Сохраняет структуру Я/Собеседни:ца и таймстемпы.
Сырой → transcript_raw.md, отредактированный → transcript.md.

### Этап 4: Анализ сессий с Claude

Суммаризация, ключевые темы, трекинг прогресса между сессиями.

### Backlog

- **Оптимизация формата аудио** (prob-20260403-004): system.wav 700 MB/час, можно сжать
- **VAD misattribution**: крупные чанки содержат обоих спикеров, граница нечёткая
- **Возобновляемая транскрипция**: сохранять промежуточные результаты
- **Автоматическая очистка**: удалять source tracks после подтверждения transcript
- **Интеграция в один запуск**: запись + streaming в одном процессе

### Критические правила

1. **Дорожки не должны разъезжаться** — никакой ротации/сегментации во время записи
2. **Source tracks не удалять** — нужны для VAD и пере-транскрипции
3. **E2e тест перед каждой реальной сессией** — запись 1 мин + транскрипция + проверка
4. **VAD gate перед моделью** — без VAD → галлюцинации на тишине
5. **Один model load** — никогда два экземпляра Qwen3-ASR
