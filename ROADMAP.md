## Roadmap

### Сейчас работает

- Запись микрофона (ffmpeg + AVFoundation)
- Запись системного аудио (ScreenCaptureKit, Swift binary)
- Три профиля: headphones-broken-mic, headphones, speaker
- Раздельные треки: mic (user) + system (remote)

- Микс обоих дорожек
- Post-hooks для автоматизаций

### Этап 2: Транскрипция через Whisper

**Цель:** после завершения записи автоматически получить один текстовый файл
с размеченными репликами (кто говорит).

**Что есть:**
- faster-whisper в `~/whisper-env` (Python 3.12, CTranslate2)
- Модель `whisper-he` в `~/models/whisper-he` — ivrit.ai файн-тюн large-v3-turbo на иврит (388ч)
- Модель `whisper-large-v3` в `~/models/whisper-large-v3` — общая (русский и др.)

**План:**
1. Скрипт `transcribe.py` — принимает `session_dir`, читает manifest.json
2. Определяет язык или берёт из аргумента (`--lang ru` / `--lang he`)
3. Выбирает модель: иврит → `whisper-he`, остальное → `whisper-large-v3`
4. Транскрибирует `system.wav` → реплики собеседника (role: remote)
5. Транскрибирует `mic.m4a` (или `mic_clean.wav` если без наушников) → реплики пользователя (role: user)
6. Сводит оба результата в один файл по таймстемпам:
   ```
   [00:00:12] Собеседник: Как вы себя чувствуете?
   [00:00:15] Я: Сегодня лучше, вчера было тяжело...
   [00:00:23] Собеседник: Расскажите подробнее...
   ```
7. Сохраняет `transcript.txt` и `transcript.json` (структурированный) в session_dir
8. Подключается как post-hook:
   ```
   ~/whisper-env/bin/python3 transcribe.py {session_dir} --lang ru
   ```

**Нюансы:**
- Два аудиофайла → два прохода Whisper → merge по временным меткам
- При merge: если реплики пересекаются по времени, приоритет у того,
  чей сегмент громче (или просто чередуем)
- Терапия на русском → оба трека через `whisper-large-v3`
- `whisper-he` — для других звонков на иврите
- Выбор модели через `--lang` или автоматически по manifest.json

### Этап 2.5: Заметка к записи

**Цель:** сразу после остановки записи предложить ввести краткое описание встречи.
Текст сохраняется в manifest.json и попадает в заголовок транскрипции.

**UX:**
```
Stopping recording...
  mic: 12.3 MB
  system: 8.7 MB

Описание (Enter чтобы пропустить): Разговор с Лешей про переезд в Берлин

Сохранено: therapy_20260328_140000/
```

**Что автоматически:**
- Дата и день недели (`Пятница, 28 марта 2026`)
- Время суток (`14:00`)
- Длительность (`47 мин`)
- Профиль/устройства

**Что вручную (опционально):**
- Свободный текст — тема, участники, контекст

**Где хранится:**
```json
// manifest.json
{
  "note": "Разговор с Лешей про переезд в Берлин",
  "date_human": "Пятница, 28 марта 2026, 14:00",
  "duration_human": "47 мин",
  ...
}
```

**Как попадёт в транскрипт:**
```
# Разговор с Лешей про переезд в Берлин
Пятница, 28 марта 2026, 14:00 — 47 мин

[00:00:03] Я: Привет, слышно?
[00:00:05] Леша: Да, привет...
```

**Реализация:** промпт в CLI после `stop()`, до запуска post-hooks.
Можно также передать через флаг: `python3 -m recorder start -l "therapy" --note "Сессия #12"`.

### Этап 3: Редактирование транскрипта через Claude

**Цель:** автоматически вычистить сырой транскрипт Whisper.

**Что делает Claude (промпт):**
- Убирает артефакты распознавания (повторы, мусор, "э-э-э")
- Исправляет пунктуацию и форматирование
- Сохраняет структуру Я:/Собеседница: и таймстемпы
- НЕ меняет смысл, НЕ перефразирует, НЕ добавляет от себя
- Помечает неразборчивые места как `[неразборчиво]`

**Реализация:**
- После `transcribe.py` запускается `claude` CLI через post-hook:
  ```
  claude -p "Отредактируй транскрипт разговора..." --input transcript.md --output transcript_edited.md
  ```
- Или через Claude Code SDK (Python):
  ```python
  from claude_code_sdk import query
  result = query(prompt=EDIT_PROMPT, input_file=transcript_path)
  ```
- Сырой транскрипт сохраняется как `transcript_raw.md`
- Отредактированный как `transcript.md`

### Этап 4: Анализ сессий с Claude

**Цель:** автоматический анализ после редактирования транскрипта.

**Возможные сценарии:**
- Суммаризация сессии с психологом
- Выделение ключевых тем и инсайтов
- Трекинг прогресса между сессиями
- Подготовка вопросов к следующей сессии

**Реализация:**
- Post-hook после редактирования вызывает Claude Code CLI
- Передаёт transcript.md + контекст из предыдущих сессий
- Результат сохраняется как `summary.md` в session_dir

### Известные проблемы

**Тихий mic-трек при записи параллельно с браузером:**
- Браузер (WebRTC) забирает управление gain микрофона, ffmpeg получает тихий сигнал
- mic.m4a записывается на уровне -57 dB (вместо нормальных -20...-30 dB)
- Whisper справляется — он сам нормализует аудио перед распознаванием
- Для прослушивания человеком нужна нормализация: `ffmpeg -af "loudnorm=I=-16:TP=-1.5:LRA=11"`
- loudnorm поднимает и шумы — нужен шумоподавитель перед нормализацией
- Возможные решения: записывать mic через Core Audio напрямую с фиксированным gain,
  или использовать ffmpeg filter `highpass=f=80,lowpass=f=8000,anlmdn` для шумоподавления

**Транскрипция параллельно с записью:**
- Начинать транскрипцию ещё в процессе записи, не дожидаясь остановки
- Возможные подходы: обрабатывать завершённые сегменты (при паузе),
  или стримить аудио в Whisper через chunked processing

### Улучшения транскрипции (backlog)

**Возобновление после прерывания:**
- При транскрибировании сохранять промежуточные результаты (`.partial.json`)
- Записывать последний обработанный timestamp для каждого трека
- При повторном запуске: проверить `.partial.json`, спросить "продолжить с [MM:SS]?"
- Whisper принимает `--initial_prompt` — можно передать последние фразы для контекста
