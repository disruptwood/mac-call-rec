## Roadmap

### Сейчас работает

- Запись микрофона (ffmpeg + AVFoundation)
- Запись системного аудио (ScreenCaptureKit, Swift binary)
- Три профиля: headphones-broken-mic, headphones, speaker
- Раздельные треки: mic (user) + system (remote)
- Эхоподавление при записи без наушников
- manifest.json с метаданными сессии
- Микс обоих дорожек
- Post-hooks для автоматизаций

### Этап 2: Транскрипция через Whisper

**Цель:** после завершения записи автоматически получить один текстовый файл
с размеченными репликами (кто говорит).

**Что есть:**
- faster-whisper в `~/whisper-env` (Python 3.12)
- Модель `whisper-large-v3` в `~/models/whisper-large-v3`

**План:**
1. Скрипт `transcribe.py` — принимает `session_dir`, находит manifest.json
2. Транскрибирует `system.wav` → реплики собеседника (role: remote)
3. Транскрибирует `mic.m4a` (или `mic_clean.wav` если без наушников) → реплики пользователя (role: user)
4. Сводит оба результата в один файл по таймстемпам:
   ```
   [00:00:12] Психолог: Как вы себя чувствуете?
   [00:00:15] Я: Сегодня лучше, вчера было тяжело...
   [00:00:23] Психолог: Расскажите подробнее...
   ```
5. Сохраняет `transcript.txt` и `transcript.json` (структурированный) в session_dir
6. Подключается как post-hook:
   ```
   ~/whisper-env/bin/python3 transcribe.py {session_dir}
   ```

**Нюансы:**
- Два аудиофайла → два прохода Whisper → merge по временным меткам
- При merge: если реплики пересекаются по времени, приоритет у того,
  чей сегмент громче (или просто чередуем)
- Языки: русский + иврит (возможно mixed), Whisper large-v3 справится

### Этап 3: Автоматизация с Claude Code

**Цель:** после сохранения транскрипта запускать Claude Code для анализа.

**Возможные сценарии:**
- Суммаризация сессии с психологом
- Выделение ключевых тем и инсайтов
- Трекинг прогресса между сессиями
- Подготовка вопросов к следующей сессии

**Реализация:**
- Post-hook после транскрипции вызывает Claude Code CLI
- Передаёт transcript.json + контекст из предыдущих сессий
- Результат сохраняется как `summary.md` в session_dir

### Приоритеты

1. **Транскрипция** — следующий шаг
2. **Claude Code автоматизация** — после того как транскрипция стабильна
3. **Улучшения записи** — по мере использования (эхоподавление, качество)
